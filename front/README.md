# 论文选题系统 - 学生端前端

## 项目结构

```
front/
├── src/
│   ├── api/
│   │   └── request.js          # Axios请求配置
│   ├── router/
│   │   └── index.js            # 路由配置
│   ├── views/
│   │   └── student/
│   │       ├── LoginView.vue           # 登录页面
│   │       ├── StudentLayout.vue       # 学生端布局
│   │       ├── TopicListView.vue       # 选题列表
│   │       ├── ApplicationView.vue     # 我的申请
│   │       ├── TeamView.vue            # 组队管理
│   │       └── ProfileView.vue         # 个人信息
│   ├── App.vue                 # 根组件
│   └── main.js                 # 入口文件

```

## 功能模块

### 1. 登录模块 (LoginView.vue)
- 学生账号登录
- 学号 + 密码验证
- 登录成功后跳转到选题列表

### 2. 选题列表 (TopicListView.vue)
- 浏览所有可选课题
- 按类别筛选
- 分页显示
- 查看选题详情和剩余名额
- 申请选题

### 3. 我的申请 (ApplicationView.vue)
- 查看已提交的申请
- 显示申请状态（待审核/已通过/已驳回）
- 取消待审核的申请

### 4. 组队管理 (TeamView.vue)
- 创建组队
- 查看我的组队信息
- 查看队员列表
- 接收和处理组队邀请
- 退出组队

### 5. 个人信息 (ProfileView.vue)
- 查看个人基本信息
- 修改手机号
- 修改密码

## 技术栈

- Vue 3 (Composition API)
- Vue Router 5
- Axios
- Vite

## API接口说明

所有接口都通过 `/api` 前缀代理到后端服务：

- 登录: `POST /api/student/auth/login`
- 选题列表: `GET /api/student/topic/list`
- 提交申请: `POST /api/student/application/submit`
- 取消申请: `DELETE /api/student/application/cancel`
- 个人信息: `GET /api/student/profile/info`
- 修改手机: `PUT /api/student/profile/phone`
- 修改密码: `PUT /api/student/auth/password`
- 创建组队: `POST /api/student/team/create`
- 组队邀请: `GET /api/student/team/invites`
- 处理邀请: `PUT /api/student/team/invite/handle`
- 退出组队: `DELETE /api/student/team/quit`

## 注意事项

1. 用户登录后，studentId 会保存在 localStorage 中
2. 所有需要认证的接口都会从 localStorage 获取 studentId
3. 样式采用简约设计风格，符合用户偏好
4. 所有组件都使用 `<script setup>` 语法
   学生端论文系统页面（主页面和顶层页面）加上返回按钮，退出按钮只在论文选题页面和队伍页面（一级页面和一层页面）显示，其余页面替换为返回按钮显示 ，返回层级相同时正常返回，返回层级不同时优先返回上层页面。
   页面按等级分为主页面和（1,2,3，等）多级页面。按照打开顺序分为顶层页面和（1,2,3，等）多层页面
   论文选题页面：
   1想要选择多人选题只有身份为队长的才可进入，根据组队状态和队伍身份提示相应信息。
   2点击多人或单人后，两按钮消失，显示选题列表页面（二级层），在顶部提示“申请选题后未确认提交保留一天名额，每天00.00刷新”且为倒计时，倒计时结束后提示已释放名额。
   3点击申请选题后，不再弹出二次提示且保留一天名额（多人队伍同样，按队伍实际申请时人数保留名额），每天00.00刷新释放名额。进入详细申请页面（3层级）最底部有一确认提交按钮。保存信息放到队伍表，缺少字段就新增。
   队伍页面：
   1点击“创建队伍”后，需填写队名（最长20字符，不可为空，不可用标点符号，输入内容只能为汉字或数字）、申请题目（不可为空）、预计人数（最多5人，最少2人，默认两人，选择题目后根据所选题目剩余可选人数提示预计人数是否可行。在同一行展示小正方形，不可行时不可点击确认创建队伍按钮且小正方形变红，可行时正方形变为绿色）。创建完成后创建队伍按钮去除，加入队伍去除。页面上半展示队伍基本信息，点击后打开详细队伍信息可操作队员（踢出，职责等），底部同一行有解散（点击条件为队伍只有创建者一人，需二次确认）和确认右上角有小叉叉按钮。创建成功后保留预计人数的名额半个小时（此保留信息可和申请选题同时进行，但此保留不受00.00刷新规则），创建成功后提示成功且提示保留30.00分钟人数名额，请尽快提交选题
   2去掉申请信息按钮，“收到的邀请”改为“信息”两字，未创建队伍和未加入队伍的人显示申请信息和邀请信息（被邀请），队长和队员显示邀请信息（主动邀请）和请求信息，信息分为邀请信息和请求请求和申请信息，可拒接或接受，接受后自动拒绝所有请求，同时更新信息状态队长和队员的组队信息显示某某某已拒绝、
   3学生接受邀请后创建队伍按钮去除，加入队伍去除。页面上半展示队伍基本信息，点击后打开详细队伍信息，底部有退出队伍（需二次确认）。不可再操作论文选题页面的两按钮，提示相应信息
